live_paid_orders ksqlDB stream：即時付費訂單流，過濾 is_real=true ✅ ksqlDB 真實處理
live_paid_orders_real live_paid_orders 的 real 過濾子集 ✅ ksqlDB
revenue_hourly ksqlDB 聚合：每小時營收統計（CTAS） ✅ ksqlDB
✅ ① 前端送出訂單
POST /api/checkout 帶 JWT Token → camping-api 驗證 JWT → 解析 userId/email
✅ ② 定價 + 優惠券
HTTP 呼叫 spot-service 取單價 → 驗證優惠券有效性 → MongoDB 原子更新 coupon.status=USED
✅ ③ 訂單寫入 MongoDB
orders 集合新增一筆，status=confirmed，含 userId/email/spotId/nights/finalTotal/couponCode
✅ ④ Kafka 非同步 produce
非同步執行 kafkaCheckoutService.send()，Avro 序列化，注入 X-Instana-T/S trace header，同步 .get() 等待確認 (~850ms)
✅ ⑤ ksqlDB 串流處理
raw_events → orders_processed → live_paid_orders → revenue_hourly，全部由 ksqlDB 自動消費處理，Instana 可見
⚠️ ⑥ order-processor（模擬）
Java consumer 讀取 raw_events，驗證欄位後輸出 LOG。
